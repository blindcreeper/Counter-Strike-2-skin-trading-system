# CS2 量化交易策略权重优化指南

## 概述

当前系统支持通过回测数据自动优化策略权重参数，包括：
- 动量因子权重 (momentum_weight)
- 平台价差权重 (spread_weight)
- 平台对权重 (pair_weight)
- 价格区间权重 (price_weight)
- 关键词权重 (keyword_weight)

## 优化算法

### 1. SciPy全局优化 (推荐)
**算法**: 差分进化算法 (Differential Evolution)
- **优势**: 全局优化，不易陷入局部最优，支持连续参数空间
- **适用**: 参数空间较大，目标函数复杂的情况
- **效率**: 通常比网格搜索高效10-100倍

### 2. 随机搜索
**算法**: 从离散参数集中随机采样
- **优势**: 简单快速，适合快速测试
- **适用**: 参数空间较小，需要快速结果的情况

### 3. 网格搜索
**算法**: 遍历所有参数组合
- **优势**: 确定性，保证找到局部最优
- **劣势**: 计算量大，效率低

## 使用方法

### 1. 准备数据
确保 `opportunities.csv` 文件包含足够的历史交易数据（建议至少100条记录）。

### 2. 运行优化
```bash
# 使用SciPy全局优化（推荐）
python run_weight_optimization.py

# 或指定其他方法
python -c "from scripts.weight_optimizer import WeightOptimizer; opt = WeightOptimizer(); opt.optimize_weights(method='random', max_evaluations=50)"
```

### 3. 查看结果
优化完成后，会显示：
- 最佳权重组合
- 对应的性能指标（胜率、平均利润、夏普比率等）
- 权重会自动保存到 `config/config.py`

### 4. 应用权重
优化后的权重会自动应用到策略中，无需手动修改。

## 权重说明

### momentum_weight (动量权重)
控制 slope/ER/Hurst/changes 对总分的影响
- **范围**: 0.4-0.9
- **影响**: 较高值注重技术指标，较低值减少对市场波动的敏感度

### spread_weight (价差权重)
控制平台价差对总分的影响
- **范围**: 0.8-1.6
- **影响**: 较高值更看重利润空间

### pair_weight (平台对权重)
控制特定平台组合的偏好
- **范围**: 0.3-1.0
- **影响**: 基于历史数据学习的平台对表现

### price_weight (价格区间权重)
控制买入价区间对总分的影响
- **范围**: 0.2-0.6
- **影响**: 避免极端价格段的交易

### keyword_weight (关键词权重)
控制物品类型对总分的影响
- **范围**: 0.4-0.8
- **影响**: 基于历史数据学习的品类表现

## 性能评估指标

### 主要指标
- **夏普比率**: 风险调整后收益，目标函数核心
- **总收益率**: 所有交易的累计收益
- **最大回撤**: 最大亏损幅度（惩罚项）

### 辅助指标
- **胜率**: 盈利交易比例
- **平均利润**: 单笔平均收益率
- **交易次数**: 策略发出的买入信号数量

## 注意事项

1. **数据质量**: 优化结果依赖于历史数据的质量和代表性
2. **过拟合风险**: 避免在小数据集上过度优化
3. **市场变化**: 定期重新优化以适应市场变化
4. **计算时间**: SciPy优化通常在几分钟内完成，网格搜索可能需要更长时间
